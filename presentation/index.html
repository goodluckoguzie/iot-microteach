Good morning panel. Today I'll demonstrate a hands-on IoT session that bridges theory to practice through a motion-detection security system. This is designed for first-year computing students with no prior hardware experience.

Learning Objectives

By the end of this session, students will be able to:

  1. Define the basic concept of Internet of Things (IoT).
  2. Identify the components of a simple sensor circuit.
  3. Construct a motion-detection logic using an Arduino microcontroller.
These are measurable outcomes. By the end, students can explain what IoT is, recognize an INPUT vs OUTPUT device, and write basic Arduino logic. Notice I've avoided jargon—accessibility is key.

What is IoT?

The network of physical objects ("things") embedded with sensors and software to connect and exchange data.

The Challenge: Home Security

[ASK] 'How would you detect an intruder using a sensor?' Excellent—motion! A PIR sensor is perfect. Our system will follow this simple logic: Sensor detects heat → Arduino checks 'motion?' → IF yes, turn on LED alarm.

Our Circuit Blueprint

Tinkercad Simulation View

  • Input:    PIR Sensor (Pin 2)
  • Output:   LED / Buzzer (Pin 13)
  • Brain:    Arduino Uno
Here's the blueprint we'll build in Tinkercad. PIR sensor on Pin 2 as INPUT, LED on Pin 13 as OUTPUT. Notice the resistor—that's to protect the LED from burning out. This is what you'll see on your screen during the demo.
